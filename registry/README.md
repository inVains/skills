# Skill Registry

This directory is the control plane for unified skill management in this repository.

## Files

- `registry.json`: Declarative source and skill inventory
- `locks.json`: Resolved refs and content hashes generated by `./skillctl lock`

## Object Model

### Source

A source is where skills come from.

Required fields:
- `id`
- `source_kind`: `managed` or `external_git`
- `fetch_strategy`: `none`, `submodule`, `detached_clone`, or `mirror_clone`
- `repo_layout`: `direct_children`, `single_skill`, or `monorepo_skills`
- `local_path`
- `origin`
- `status`

Additional fields:
- `source_url`
- `source_ref`
- `skills_root`
- `include_paths`

### Skill

A skill is the unit agents enable, inspect, export, and restore.

Required fields:
- `id`
- `source_id`
- `relative_path`
- `enabled`
- `license_class`
- `export_policy`

Additional fields:
- `runtime`
- `dependency_files`
- `entry_files`
- `tags`

## CLI

Read commands emit stable JSON:

```bash
./skillctl source list --json
./skillctl source inspect anthropic-local --json
./skillctl source scan anthropic-local --json
./skillctl skill list --json
./skillctl skill inspect skill-manager --json
./skillctl sync --json
./skillctl doctor --json
./skillctl lock --json
```

Mutating commands:

```bash
./skillctl source add my-source --url https://example.com/skills.git --layout monorepo_skills --skills-root skills --json
./skillctl source scan my-source --register --json
./skillctl source update my-source --json
./skillctl source remove my-source --force --json
./skillctl skill disable skill-manager --json
./skillctl skill enable skill-manager --json
./skillctl export --skills skill-manager config-safe --json
./skillctl restore exports/<bundle-name> --json
```

Dependency installation is safe by default: `./skillctl install <skill> --json`
only returns an install plan. Use `--execute` to run package manager commands.

## External Monorepos

For repositories like `anthropics/skills`, register the repository as one
source and the individual skill folders as skills:

```bash
./skillctl source add anthropic-upstream \
  --url https://github.com/anthropics/skills.git \
  --layout monorepo_skills \
  --skills-root skills \
  --origin anthropic \
  --json
./skillctl source scan anthropic-upstream --json
./skillctl source scan anthropic-upstream --register --disabled --json
```

This keeps the repository-level source separate from skill-level enablement.
